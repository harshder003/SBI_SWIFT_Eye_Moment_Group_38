# SWIFT Model of Eye Movements - Simulation-Based Inference

This project aims to implement a simplified version of the SWIFT model, a dynamic generative model of eye movement control during reading (Engbert & Rabe, 2024).

## Project Overview

In reading research, understanding how readers decide when to move their eyes (fixation duration) and where to look next (saccade target) is a key question. The SWIFT model simulates these dynamics. Because the full version of SWIFT is computationally intensive and has an intractable likelihood, we use **Simulation-Based Inference (SBI)**.

Specifically, we use the `BayesFlow` library to train a neural network on synthetic data generated by a simplified SWIFT simulator. Once trained, the network can perform rapid amortized inference to estimate the cognitive parameters of individual human readers based on their eye-tracking data.

## Workflow

1.  **Simulation Phase:** We build a generative model (the simulator) that mimics the simplified SWIFT model. It takes cognitive parameters (e.g., base fixation duration, saccade probability) and generates sequences of fixations and landing positions.
2.  **Training Phase:** We use `BayesFlow` to learn the mapping from simulated eye-tracking summary statistics back to the underlying parameters.
3.  **Inference Phase:** We apply the trained network to real human eye-tracking data (`input/fixseqin_PB2expVP10.dat` and `input/Rcorpus_PB2.dat`) to obtain the posterior distributions for the reader's cognitive parameters.

## Directory Structure
- `input/`: Contains the `.dat` data files for the corpus and the individual participant's fixation sequence.
- `visualizations/`: Contains EDA plots showing distributions of word lengths, frequencies, and empirical fixation behaviors.
- `eda_script.py`: Script to generate summary statistics and visualizations.
- `swift_simulator.py`: Python script containing the generative simulator for BayesFlow.
